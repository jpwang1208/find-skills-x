# Skill Quality Assessment

Skill质量评估标准和流程，确保推荐的skill可靠、安全、适合。

---

## 质量评估框架

### 核心指标

| 指标 | 权重 | 说明 | 数据来源 |
|------|------|------|----------|
| **安装量** | 40% | 使用广度 | skills.sh |
| **Stars** | 20% | 社区认可 | GitHub |
| **更新频率** | 20% | 维护活跃度 | GitHub commits |
| **源信誉度** | 20% | 作者可信度 | 作者身份 |

### 评分标准

#### 安装量分数 (40%)

| 范围 | 分数 | 等级 |
|------|------|------|
| >= 100,000 | 5 | 极高 |
| >= 10,000 | 4 | 高 |
| >= 1,000 | 3 | 中 |
| >= 100 | 2 | 低 |
| < 100 | 1 | 极低 |

#### GitHub Stars分数 (20%)

| 范围 | 分数 | 等级 |
|------|------|------|
| >= 500 | 5 | 极高 |
| >= 100 | 4 | 高 |
| >= 50 | 3 | 中 |
| >= 10 | 2 | 低 |
| < 10 | 1 | 极低 |

#### 更新频率分数 (20%)

| 时间范围 | 分数 | 等级 |
|----------|------|------|
| 1个月内 | 5 | 极活跃 |
| 3个月内 | 4 | 活跃 |
| 6个月内 | 3 | 正常 |
| 1年内 | 2 | 不活跃 |
| > 1年 | 1 | 停滞 |

#### 源信誉度分数 (20%)

| 来源类型 | 分数 | 示例 |
|----------|------|------|
| 官方组织 | 5 | anthropics, vercel-labs, microsoft |
| 知名公司/团队 | 4 | expo, sentry, langchain |
| 活跃贡献者 | 3 | 经常维护的社区作者 |
| 普通用户 | 2 | 个人开发者 |
| 未知 | 1 | 无法确认身份 |

---

## 计算公式

```
质量分 = (安装量分 × 0.4) + (Stars分 × 0.2) + (更新分 × 0.2) + (信誉分 × 0.2)
```

### 质量分等级

| 分数范围 | 等级 | 推荐度 |
|----------|------|--------|
| 4.5 - 5.0 | 优秀 | ⭐⭐⭐ 强烈推荐 |
| 3.5 - 4.4 | 良好 | ⭐⭐ 推荐 |
| 2.5 - 3.4 | 一般 | ⭐ 可用 |
| 1.5 - 2.4 | 较差 | ⚠️ 谨慎 |
| 1.0 - 1.4 | 极差 | ❌ 不推荐 |

## 评估流程

### Step 1: 收集基础数据

```bash
# 获取安装量
npx skills find <skill-name>  # 查看输出中的安装量

# 获取GitHub信息
gh repo view <owner/repo> --json stargazerCount,updatedAt

# 获取最近提交
gh api repos/<owner>/<repo>/commits --paginate -q '.[0:5].commit.author.date'
```

### Step 2: 计算各项分数

```javascript
// 示例计算
const skill = {
  installs: 8500,      // -> 4分 (>=1K)
  stars: 230,          // -> 4分 (>=100)
  lastUpdate: "2024-01-15",  // 3个月内 -> 4分
  source: "vercel-labs"      // 知名公司 -> 4分
};

const qualityScore = 
  (4 * 0.4) +  // 安装量
  (4 * 0.2) +  // Stars
  (4 * 0.2) +  // 更新
  (4 * 0.2);   // 信誉
// = 4.0 (良好)
```

### Step 3: 风险检查

**自动检查项**：
- [ ] 安装量 >= 100
- [ ] 6个月内有更新
- [ ] 来源可验证
- [ ] 无已知安全问题

**手动检查项**：
- [ ] SKILL.md内容合理
- [ ] 脚本无恶意代码
- [ ] 依赖无漏洞

### Step 4: 生成报告

```markdown
## Skill质量报告: {skill-name}

### 基础数据
| 指标 | 值 | 分数 |
|------|-----|------|
| 安装量 | 8,500 | 4/5 |
| Stars | 230 | 4/5 |
| 更新时间 | 2024-01-15 | 4/5 |
| 来源 | vercel-labs | 4/5 |

### 综合评分
**质量分: 4.0/5 (良好)**

### 风险评估
✅ 安装量充足
✅ 维护活跃
✅ 来源可信
✅ 无已知风险

### 推荐意见
推荐安装。适合{使用场景}。
```

## 风险评估

### 高风险信号 ⚠️

出现以下任一情况，需要特别警告用户：

| 风险信号 | 说明 | 处理建议 |
|----------|------|----------|
| 安装量 < 100 | 使用者少，未经充分验证 | 标注风险，建议测试 |
| 6个月无更新 | 可能已停止维护 | 警告可能不兼容新版本 |
| 来源不明 | 无法验证作者身份 | 建议审查代码 |
| 安全报告 | 有已知漏洞或问题 | 不推荐 |
| 恶意代码特征 | 脚本包含可疑操作 | 禁止推荐 |

### 中风险信号 ⚡

需要提醒用户注意：

| 风险信号 | 说明 |
|----------|------|
| 依赖过时 | 依赖包版本老旧 |
| 文档不完整 | 缺少使用说明 |
| 兼容性限制 | 只支持特定环境 |
| 性能问题 | 已知性能瓶颈 |

### 低风险信号 ✓

通常可以接受：

| 情况 | 说明 |
|------|------|
| 新发布的skill | 刚发布，安装量低但可能质量好 |
| 小众领域 | 用户少但专注 |
| 个人项目 | 个人维护但活跃 |

## 特殊评估

### 新发布skill评估

对于发布时间 < 30天的skill：

1. **降低安装量权重**：从40%降到20%
2. **增加来源权重**：从20%升到30%
3. **增加内容审查**：手动检查SKILL.md
4. **标注状态**：明确告知用户新发布

### 企业/私有skill评估

对于非公开skill：

1. **忽略公开指标**：安装量、Stars不适用
2. **内部验证**：检查内部审核状态
3. **安全审查**：确保符合企业规范
4. **文档完整**：确保有完整使用文档

### 中文/本地化skill评估

对于特定语言/地区的skill：

1. **语言匹配**：确认支持用户语言
2. **本地化质量**：检查翻译/本地化质量
3. **区域适用**：确认适用于用户所在地区

## 持续监控

### 监控指标

定期检查已推荐skill的状态：

- 安装量变化趋势
- Stars增长情况
- 最近更新时间
- Issue处理情况

### 质量下降处理

当已推荐skill质量下降：

1. 更新评估报告
2. 调整推荐等级
3. 通知用户
4. 考虑替代方案
